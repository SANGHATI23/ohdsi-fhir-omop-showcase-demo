Framework comparator analysis. To strengthen the flattening comparison beyond naive JSON expansion, we added a local-file comparator against FHIRpack using the same MIMIC-IV Demo on FHIR NDJSON resources. Across 16 compressed NDJSON files containing 858,025 FHIR resources, all three methods processed the same resource count. Generic pandas JSON normalization produced 195 total columns across files, FHIRpack Frame construction produced 171 columns, and compact deterministic FHIRy-style flattening produced 150 columns. Thus, the compact FHIRy representation used 23.1% fewer columns than generic JSON normalization and 12.3% fewer columns than FHIRpack Frame output. Runtime differed by method: FHIRpack Frame construction was fastest in this local benchmark at 2.49 seconds total, followed by compact FHIRy-style flattening at 14.36 seconds and generic JSON normalization at 17.19 seconds. These results support the compactness of the proposed analytics-oriented representation while showing that FHIRpack provides faster local Frame construction. The comparator should be interpreted as a shared-input local tabularization benchmark rather than a full production ETL, server-query, or vocabulary-standardization benchmark.
